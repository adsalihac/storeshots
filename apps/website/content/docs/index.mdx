Storeshots turns the repetitive store screenshot workflow into a single command for Expo apps.

```bash
npx storeshots
```

It discovers Expo Router routes, captures screenshots for supported device presets, processes the images, and writes a predictable output folder that is ready for App Store Connect and Google Play Console.

## What Storeshots solves

Mobile releases often require the same manual loop: open a simulator, navigate every release screen, take screenshots, resize them, and repeat for each device size. Storeshots moves that work into a repeatable CLI pipeline.

## Core workflow

1. Discover routes from `app/` or `src/app/`.
2. Capture each route for selected Apple and Android devices.
3. Resize, crop, and export high quality PNG assets.
4. Commit or upload the generated screenshots in CI.

## Monorepo packages

- `@storeshots/cli` exposes the `storeshots` binary.
- `@storeshots/core` coordinates generation.
- `@storeshots/expo-router-discovery` maps files to routes.
- `@storeshots/screenshot-engine` owns device capture.
- `@storeshots/image-processor` exports store-ready PNGs.
- `@storeshots/device-presets` defines Apple and Android sizes.
